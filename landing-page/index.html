Introduction

You have a limited number of keystrokes left in your life. Do you really want to spend them configuring Babel?

Every time you run npm install, you create a black hole of dependencies you don't understand and will never audit. You feel the gap in your knowledge—the information gap—and it hurts. You scratch the itch by adding another abstraction layer. It doesn't help.

boreDOM is the release valve.

It is a Single-File Runtime. Logic, UI, and Styles live in one .html file. It eliminates context switching. It stops the "hallucination loop" where LLMs forget your file structure. It runs in the browser. It is ugly. It works.

Code

Copy this. Paste it. It runs. 


<!-- 1. The Component -->
<simple-counter></simple-counter>

<!-- 2. The Template -->
<template data-component="simple-counter">
  <div>
    <span data-text="local.count"></span>
    <button data-dispatch="inc">+</button>
  </div>
</template>

<!-- 3. The Logic -->
<script type="text/boredom" data-component="simple-counter">
  export default ({ on, local }) => {
    local.count = 0;
    on("inc", () => local.count++);
  };
</script>

<!-- 4. The Runtime -->
<script src="boreDOM.js"></script>

The Basics

Delete package.json. The most secure dependency is the one you don't have.

No "Build". The browser is the compiler. It has been for 30 years. We just forgot.

IKEA Effect. You will value this code more because you wrote it in a single file like a caveman.
